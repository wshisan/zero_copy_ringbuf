# zero_copy_ringbuf

一个**零拷贝、无锁、线程安全**的环形队列实现，专为**同一进程内两个线程间传输大数据**设计（例如从采集线程传递传感器数据到处理线程）。通过共享内存 + 槽位预留机制，避免任何 `memcpy` 或赋值拷贝，显著降低 CPU 负载和延迟。

## 特性

- ✅ **零拷贝**：生产者和消费者直接读写共享内存中的槽位，无需额外数据拷贝。
- ✅ **无锁**：使用原子操作维护读写索引，单生产者单消费者场景无锁。
- ✅ **线程安全**：生产者和消费者可以并发运行，不会出现数据竞争。
- ✅ **泛型支持**：支持任意 POD 类型的数据块（如障碍物列表、图像元数据、CAN 信号组等）。
- ✅ **可扩展**：稍作修改即可从线程间通信扩展为进程间通信（将 `MAP_ANONYMOUS` 改为具名共享内存）。
- ✅ **跨平台**：基于 POSIX 标准（Linux / Unix），Windows 下可类似实现。
